Strategy Configuration and Development

This part of the project focuses on how strategies are set up and managed. You can work with built-in examples, tweak templates, or plug in your own logic, all while staying inside the same risk framework.

Why Use TopstepJâ€™s Strategy System?

Every strategy in TopstepJ runs through the same risk core. That means position size rules, daily loss limits, and session filters are always checked, no matter what style of logic you are using.

You can start from simple ideas like moving average crossovers or go into Smart Money Concepts and machine learning models. The main point is that the framework handles the plumbing and risk checks so you can focus on the strategy itself.

Strategy Configuration Guide

Overview

Each strategy in TopstepJ is configured through a JSON or YAML configuration file that defines parameters, risk settings, entry/exit logic, and analytics preferences. The platform loads these at startup and initializes each strategy with its own isolated state and risk context.

Example: Strategy configuration screen with parameter inputs, risk settings, and validation

Example Strategy Parameters

A typical moving average crossover strategy might include the following parameters: 

{
  "strategyName": "MovingAverageCrossover",
  "enabled": true,
  "parameters": {
    "fastPeriod": 10,
    "slowPeriod": 50,
    "instrument": "ES",
    "timeframe": "5m",
    "entrySignalThreshold": 0.75
  },
  "riskSettings": {
    "maxPositionSize": 2,
    "stopLossPercent": 0.5,
    "takeProfitPercent": 1.0,
    "maxDailyLoss": 500,
    "sessionFilters": ["RTH"]
  },
  "analytics": {
    "enablePerformanceTracking": true,
    "logSignals": true,
    "recordEquityCurve": true
  }
}

Parameter configuration panel for moving average crossover strategy

Risk Settings Configuration

Risk settings are mandatory for every strategy and get enforced by the platform's risk middleware:

  • Max Position Size: Maximum number of contracts the strategy can hold at once
  • Stop Loss Percent: Automatic stop loss percentage relative to entry price
  • Take Profit Percent: Automatic take profit percentage relative to entry price
  • Max Daily Loss: Maximum dollar loss allowed per day before strategy is disabled
  • Session Filters: Allowed trading sessions (e.g., RTH, ETH, Asian session)

Strategy Features & Architecture

Entry & Exit Logic

Each strategy follows the same general pattern for generating signals, getting into trades, and getting out. That keeps behavior consistent and makes it easier for the platform to apply risk rules in the same way everywhere.

Entry Logic Flow:

  1. Market Data Event: Strategy receives normalized market data (price, volume, indicators)
  2. Signal Generation: Strategy applies its logic to generate buy/sell signals with confidence scores
  3. Risk Validation: Signal is passed to risk middleware for position size, daily loss, and session validation
  4. Order Execution: If approved, order is routed to broker with attached stop loss and take profit levels
  5. Position Tracking: Position is added to strategy's active position tracking and analytics

Exit Logic Flow:

  1. Exit Signal Detection: Strategy monitors for exit conditions (target hit, stop hit, time-based, reversal signal)
  2. Risk-Adjusted Exit: Risk middleware can override exit if trailing stops or time filters require earlier exit
  3. Order Execution: Exit order is sent to broker and position is closed
  4. Performance Recording: Trade outcome (P&L, duration, slippage) is recorded in analytics database
  5. State Update: Strategy state is updated and made available for analysis and model retraining

Risk Hooks & Analytics Per Strategy

Each strategy has its own risk context and its own stats. That lets you compare them side by side and see which ideas work better in different types of markets.

Risk Hooks:

  • Pre-Entry Hook: Validates position size, daily loss limit, and available margin before entry
  • Position Monitoring Hook: Continuously monitors open positions against stop loss, take profit, and trailing stop levels
  • Session Filter Hook: Automatically closes positions and prevents new entries outside allowed trading sessions
  • Drawdown Circuit Breaker: Disables strategy if daily or weekly drawdown limits are exceeded

Per-Strategy Analytics:

  • Equity Curve: Real-time equity tracking showing cumulative P&L over time
  • Win Rate & Risk/Reward: Win rate percentage, average winner vs. average loser, profit factor
  • Drawdown Analysis: Maximum drawdown, current drawdown, recovery time
  • Signal Quality Metrics: Signal accuracy, false positive rate, signal confidence distribution
  • Performance by Market Condition: Performance segmented by volatility, trend, and time of day
