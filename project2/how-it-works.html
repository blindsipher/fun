How TopstepJ Works

TopstepJ uses an event-driven setup that pushes each piece of market data through the same pipeline. Before anything turns into an order, it has to pass the risk checks first.

Event-Driven Architecture

Instead of a pile of one-off scripts or manual clicks, TopstepJ treats everything as events flowing through a pipeline. Market data comes in, moves through each stage in order, and you always know what step it’s at.

The architecture splits things like data ingestion, risk checks, strategy logic, and order routing into their own components. That makes it easier to log, debug, and tune each part without breaking the rest.

The goal is a platform that is safer and more disciplined to use, but also easier to understand and tweak if you are the one building or testing strategies.

The Event Pipeline

Here is the basic path each market event follows in TopstepJ, from the moment data comes in to the point where you see fills and analytics.

1

Ingestion of Market Data

Market data (tick data, order book updates, trade executions) comes in from broker APIs, data feeds, or simulation sources. This raw data gets captured and queued for processing.

Key components: Data connectors, message queues, raw event streams

2

Normalization

Raw market data gets normalized into a standardized format. Different brokers and data sources have different schemas—normalization makes sure all downstream components work with consistent data structures.

Key components: Data transformers, schema validators, timestamp synchronization

3

Risk Middleware

Before any strategy logic runs, normalized data passes through the risk middleware layer. This is where we enforce account limits, position sizing rules, daily loss limits, and session filters. If you violate risk rules, the event gets flagged or halted.

Key components: Risk engine, position tracking, loss limit monitors, time filters

4

Strategy Plugins and Signal Generation

Events that pass risk validation go to active strategy plugins. Each strategy analyzes the data, applies its logic (technical indicators, SMC patterns, ML model predictions), and generates trading signals (buy, sell, hold).

Key components: Strategy manager, plugin system, signal generators, SMC and ML modules

5

Order Routing

Valid trading signals get converted into orders and routed to the broker or execution venue. The order router handles order formatting, submission, and acknowledgment tracking.

Key components: Order manager, broker API integrations, execution tracking

6

Analytics and Feedback Loops

After execution, trade results get captured and analyzed. Performance metrics (P&L, win rate, drawdown) are calculated per strategy and per account. This data feeds back into the system for optimization and reporting.

Key components: Analytics engine, performance tracking, reporting dashboards, feedback loops

It shows how data flows from ingestion through normalization, risk checks, strategy execution, order routing, and analytics feedback.

  • Risk Core: Enforces trading limits and risk rules
  • Middleware: Coordinates between components
  • Strategy Plugins: Modular, extensible strategy system
  • SMC/ML Components: Smart Money Concepts and machine learning integrations
  • Strategy & Account Analytics: Real-time performance monitoring and reporting
