Demo: Getting TopstepJ Up and Running

This page walks through what you would need to install, configure, and run TopstepJ, plus a look at the interface once it is open.

System Requirements

Before installing TopstepJ, it helps to know what kind of system it expects so you are not fighting performance issues the whole time.

Supported Operating Systems

  • Linux: Ubuntu 20.04+, Debian 11+, Fedora 35+, or equivalent distributions (64-bit)

Software Dependencies

  • Git: Version 2.30+ (for cloning the repository)
  • Java Runtime: JDK 17 or later
  • Node.js: Version 16.x or later (for web dashboard)
  • PostgreSQL: Version 13+ (for data persistence)
  • Redis: Version 6+ (for caching and message queuing)

Hardware Requirements

  • Processor: Dual-core CPU, 2.5 GHz or faster (I'd recommend quad-core for live trading)
  • Memory: Minimum 16 GB RAM (32 GB recommended if you're running multiple strategies)
  • Storage: At least 20 GB free disk space for app data and historical market data
  • Network: Stable internet connection with low latency (this is essential for live market data)

Platform Interface

After installation, the main thing you see is the trading interface: account info on one side, order entry on the other, and controls for how you want trades to be sent.

TopstepJ Platform Demo: Watch the platform in action with real time trading

Key Interface Components

  • Accounts Panel: Live view of account balances, IDs, and trade counts so you always know where the account sits
  • Manual Order Entry: Order ticket for picking contracts, choosing order types (Market, Limit, Stop), and setting up bracket orders
  • Risk Management Parameters: Built in controls for position sizing, stop loss placement, and take profit targets—disciplined risk management on every trade
  • Trailing Stop Loss: Trailing stop options with adjustable distances to help lock in profits while trades are moving
  • Quick Execution Controls: Intuitive buy and sell buttons with real time order preview and confirmation for rapid execution when opportunities arise

Troubleshooting & Tips

Common issues and best practices for running TopstepJ on Linux.

Permissions and Security

  • Set proper file permissions: chmod -R 755 topstepj/
  • Make sure PostgreSQL and Redis services are enabled to start on boot
  • Configure firewall rules using ufw or iptables if you need to
  • Use environment variables for sensitive credentials—don't hardcode them

Common Issues

  • Port conflicts: Make sure ports 5173 (frontend) and 8000 (backend) aren't being used by other applications. Check with sudo lsof -i :5173 and sudo lsof -i :8000
  • Frontend not loading: Verify the Vite dev server is running on port 5173 and check the console for errors
  • API connection errors: Make sure the backend server is running on port 8000 and both servers are started
  • Database connection errors: Verify PostgreSQL is running with systemctl status postgresql
  • Redis connection errors: Verify Redis is running with systemctl status redis
  • Out of memory errors: Monitor system resources with htop and make sure you meet the minimum 16GB RAM requirement

Performance Optimization

  • Enable PostgreSQL connection pooling for better database performance
  • Configure Redis maxmemory settings based on available system RAM
  • Use pm2 process manager for automatic restart and monitoring
  • Monitor system resources regularly using htop, iotop, and nethogs
  • Consider setting up log rotation to prevent disk space issues
